📖 Jak działa autentykacja

Architektura v2.0 — Supabase + GitHub

Twoje dane w Supabase:
• Konfiguracja GitHub (owner, repo, token)
• Punkt startowy bankrollu
• Wszystkie transakcje (stawki + wypłaty)
• Każde konto widzi tylko swoje dane (Row Level Security)

GitHub — tylko odczyt:
• Kupony z data/results/coupons_history.json
• PAT potrzebuje tylko scope repo (read)
• Brak zapisów do repo → brak błędu 403

GitHub Secrets (static.yml):
SUPABASE_URL — URL Twojego projektu Supabase
SUPABASE_ANON_KEY — klucz publiczny (anon)

Oba klucze są wstrzykiwane podczas deploymentu GitHub Pages i nigdy nie trafiają do kodu źródłowego.
